# MukMap 디자인 문서

> **stitch 프로젝트**: ID `18166411566732928731`
> 스크린샷: `spec-docs/design-assets/` 디렉토리에 저장

## 1. 디자인 원칙

- **지도 중심**: 지도가 메인. UI는 지도를 가리지 않되 필요할 때 접근 가능.
- **정보 점진적 노출**: 마커(위치) → 인포윈도우(요약) → 상세 페이지(전체) 3단계.
- **톤앤매너**: 따뜻하고 맛있는 느낌. 오렌지 + 화이트 기반. 깔끔하되 차갑지 않게.
- **참고 서비스**: 카카오맵 (지도 UI), 망고플레이트 (맛집 카드), 왓챠피디아 (필터 UX)

- **진입 경로**: mukmap.kr 직접 접속

```
│ 유튜버 필터   │                                      │
│ ┌──────────┐ │                                      │

│ │⚪ 백종원  55│ │        📍    📍📍                    │
│ │⚪ 돌시네아 23││            ┌──────────┐              │
│ └──────────┘ │            │ 성게향     │              │

│ [한식][일식]  │            └──────────┘              │
```

- **상단 바**: 높이 56px, 배경 white, 하단 그림자 `shadow-sm`
  - 로고: 오렌지 아이콘 + "MukMap" Bold 20px
  - 검색바: 너비 400px, 높이 40px, 둥근 모서리 8px, 회색 보더
- **사이드 패널**: 너비 320px, 배경 Gray 50, 오른쪽 보더
  - 유튜버 목록: 아바타 40px 원형 + 이름 + 맛집 수 배지
  - 선택 시: 배경 Primary Light + 좌측 3px Primary 보더
  - 지역 드롭다운: 전체 너비, 높이 36px
  - 카테고리 칩: 높이 32px, 둥근 모서리 16px, 비활성 Gray 200 / 활성 Primary
- **지도**: 나머지 영역 전체
- **인포윈도우**: 너비 280px, 배경 white, 둥근 모서리 12px, 그림자
  - 가게명: H3 (18px Bold)
  - 카테고리·지역: Caption (12px Gray 600)
  - 유튜버: 아바타 24px + 이름 + 평가 배지
  - 버튼: "상세 보기" (Primary), "네이버 지도" (outline), "영상 보기" (outline)

**상태별 디자인**:
- **정상**: 위 레이아웃
- **빈 상태**: 지도 중앙에 일러스트 + "이 지역에는 아직 수집된 맛집이 없어요" + 인기 지역 버튼 4개 (서울/부산/전주/제주)
- **로딩**: 사이드 패널 스켈레톤 (유튜버 목록 5줄 회색 박스), 지도 영역 반투명 오버레이 + 스피너
- **에러**: 지도 영역에 에러 아이콘 + "지도를 불러올 수 없습니다" + 새로고침 버튼

- **목적**: 모바일에서 지도 탐색

**레이아웃**:
```
┌─────────────────────┐
│ [🍊] [🔍 검색... ] │  ← 상단 바 (48px)

│     (전체 화면)      │
│                     │
│   📍  📍            │

│ ━━━━ (핸들)          │
```

- 키보드: ↑↓ 네비게이션, Enter 선택, ESC 닫기

### 4.6-4.8 Empty / Error / Loading 상태

**Empty State** (필터 결과 없음):
- 중앙 정렬 일러스트 (빈 접시 아이콘)
- "조건에 맞는 맛집이 없어요"  Body 1, Gray 600
- "필터를 조정해보세요" Caption, Gray 400
- [필터 초기화] Primary 버튼

**Error State**:
- 네트워크: WiFi 끊김 아이콘 + "인터넷 연결을 확인해주세요"
- 서버 에러: 느낌표 아이콘 + "서버에 문제가 발생했습니다" + [다시 시도] 버튼
- 404: "이 페이지를 찾을 수 없어요" + [메인으로] 버튼

**Loading State**:
- 지도: 반투명 오버레이 + 오렌지 스피너
- 리스트: 스켈레톤 카드 (회색 박스 펄스 애니메이션)
- 검색: 드롭다운 내 3줄 스켈레톤

## 5. 네비게이션 플로우

```
메인 지도 ←──────────────────────────────────→ 검색 자동완성
  │                                              │
  ├── 마커 클릭 → 인포윈도우                        │
  │     ├── "상세 보기" → 맛집 상세 페이지           │
  │     ├── "네이버 지도" → 외부 (새 탭)            │
  │     └── "영상 보기" → 유튜브 (새 탭)            │
  │                                              │
  ├── 유튜버 클릭 → 유튜버 프로필 페이지 ←────────┘
  │     └── 맛집 카드 클릭 → 맛집 상세 페이지
  │
  └── 검색 결과 선택
        ├── 맛집 → 지도 이동 + 인포윈도우
        ├── 유튜버 → 유튜버 필터 적용
        └── 지역 → 지역 필터 적용

맛집 상세 페이지
  ├── "네이버 지도에서 보기" → 외부 (새 탭)
  ├── 유튜버 아바타 클릭 → 유튜버 프로필 페이지
  └── [← 뒤로] → 이전 페이지
```

## 6. 인터랙션 정의

| 인터랙션 | 동작 |
|----------|------|
| 마커 클릭 | 인포윈도우 열림 (기존 열린 것은 닫힘). 부드러운 페이드인 150ms |
| 지도 이동 | 이동 멈춤 후 300ms debounce → API 재요청 → 마커 업데이트 |
| 유튜버 필터 토글 | 마커 즉시 표시/숨김. 지도 bounds fit 애니메이션 500ms |
| 카테고리 칩 토글 | 선택 시 배경 Primary + 흰색 텍스트. 해제 시 원복. 전환 150ms |
| 검색 자동완성 | 디바운스 300ms. 드롭다운 슬라이드다운 200ms |
| 토스트 | 하단 중앙에서 슬라이드업. 3초 후 자동 사라짐. 닫기(X) 가능 |
| 모바일 하단 시트 | 드래그로 3단계 전환. 스프링 애니메이션. 배경 터치 시 COLLAPSED |
| 무한 스크롤 | 하단 80% 도달 시 다음 페이지 로드. 로딩 중 스피너 |
| 인포윈도우 → 상세 | 페이지 전환 (Next.js Link). 뒤로가기 시 지도 상태 유지 |
