# MukMap PRD

## 1. 개요

- **서비스 한줄 설명**: 먹방 유튜버의 영상 자막에서 맛집을 자동 추출하여 지도에 표시하는 웹 서비스
- **해결하려는 문제**: 맛집 유튜버 영상을 보고 "거기 어디야?"라고 검색하는 수고. 영상마다 가게 이름·위치를 일일이 찾아야 하고, 여행 계획 시 한 유튜버가 다녀간 맛집을 한번에 보기 어려움.
- **타겟 유저 페르소나**:
  - **A. 맛집 영상 시청자 (25-35세 직장인)**: 퇴근 후 유튜브 맛집 영상을 즐김. "이 유튜버가 갔던 가게 전체 목록"을 한눈에 보고 싶음. 주말에 친구와 갈 곳을 정할 때 활용.
  - **B. 여행 계획자 (20-40세)**: 전주/제주 여행을 앞두고 해당 지역 유튜버 추천 맛집을 지도 위에서 동선으로 확인하고 싶음.
  - **C. 유튜버 팬 (전 연령)**: 특정 유튜버(성시경, 쯔양 등)의 모든 맛집을 아카이빙하고 싶은 팬.

## 2. 핵심 가치 제안

| 기존 대안 | 문제점 | MukMap |
|-----------|--------|--------|
| Eatlect | 해외 서비스. 한국 유튜버 미지원 | 한국 먹방 유튜버 전문 |
| TVFoodMaps | 미국 TV 프로그램 기반 | 유튜브 자막 기반 자동 수집 |
| 수동 검색 | 영상 → 더보기 → 네이버 검색. 가게당 2-3분 | 자동. 0초 |
| 구글 마이맵 개인 제작 | 수동 업데이트, 유튜버 1명 한정 | 다채널 자동 업데이트 |

## 3. 기능 명세

### 3.1 지도 맛집 탐색 (메인)

- **설명**: 카카오맵 위에 유튜버 맛집 마커를 표시. 마커 클릭 시 가게 정보 + 영상 링크.
- **왜 필요**: 서비스의 핵심. 이게 없으면 서비스가 아님.
- **유저 시나리오**:
  1. 유저가 mukmap.kr 접속
  2. 서울 중심 카카오맵 표시, 맛집 마커 클러스터링
  3. 줌인하면 개별 마커로 펼쳐짐
  4. 마커 클릭 → 인포윈도우: 가게명, 카테고리, 주소, 유튜버 아바타, 평가
  5. "영상 보기" → 유튜브 영상으로 이동 (새 탭)
  6. "네이버 지도" → 네이버 지도 해당 가게 페이지로 이동 (새 탭)
- **입력**: 지도 조작 (드래그, 줌, 마커 클릭)
- **출력**: 마커 + 인포윈도우
- **비즈니스 규칙**:
  - 지도 뷰포트 내 맛집만 로드 (최대 200개)
  - 마커 50개 이상 시 자동 클러스터링
  - 기본 줌: 서울 전체 (level 8)
  - 기본 중심: 서울시청 (37.5665, 126.9780)
  - 좌표 없는 맛집은 지도에 표시하지 않음
- **예외 케이스**:
  - 해당 영역에 맛집 없음 → "이 지역에는 아직 수집된 맛집이 없어요" + 인기 지역 버튼 (서울/부산/전주/제주)
  - 카카오맵 SDK 로딩 실패 → "지도를 불러올 수 없습니다. 새로고침해주세요" 에러 + 새로고침 버튼
  - 네트워크 에러 → "인터넷 연결을 확인해주세요" 토스트 (3초 후 자동 닫힘)
  - 로딩 중 → 지도 영역 스켈레톤 + "맛집 불러오는 중..." 스피너
  - API 응답 지연 (3초 초과) → 스피너 유지, 10초 초과 시 "응답이 느립니다. 다시 시도해주세요" 토스트
- **우선순위**: P0

### 3.2 유튜버별 필터

- **설명**: 특정 유튜버 선택 시 해당 유튜버 맛집만 지도에 표시
- **왜 필요**: "성시경이 간 곳만 보고 싶어" — 가장 자연스러운 사용 패턴
- **유저 시나리오**:
  1. 상단 또는 사이드에 유튜버 목록 (아바타 + 이름)
  2. 유튜버 클릭 → 해당 유튜버 맛집만 필터, 지도가 해당 맛집들이 보이도록 자동 줌
  3. 복수 선택 가능 (토글)
  4. "전체" 버튼으로 필터 리셋
- **입력**: 유튜버 선택/해제 (클릭)
- **출력**: 필터된 마커, 지도 자동 줌/이동
- **비즈니스 규칙**:
  - 유튜버는 맛집 수 기준 내림차순 정렬
  - 최대 동시 선택: 5명
  - 유튜버별 고유 마커 색상 (5색 순환: #FF6B35, #2196F3, #4CAF50, #9C27B0, #FF9800)
  - 선택된 유튜버 칩에 맛집 수 배지 표시 (예: "성시경 (47)")
- **예외 케이스**:
  - 유튜버 목록 로딩 실패 → "유튜버 목록을 불러올 수 없습니다" + 재시도 버튼
  - 선택한 유튜버 맛집이 현재 뷰포트에 없을 때 → 자동으로 bounds fit (전체 마커가 보이도록 줌 조절)
  - 6명 이상 선택 시도 → "최대 5명까지 선택할 수 있어요" 토스트
  - 유튜버 데이터 0개 (아직 수집 안 됨) → 유튜버 목록에서 숨김
- **우선순위**: P0

### 3.3 지역/카테고리 필터

- **설명**: 지역(시/도)과 음식 카테고리로 맛집 필터링
- **왜 필요**: "전주 한식만 보고 싶어" — 여행 계획 시 필수
- **유저 시나리오**:
  1. 필터 바에서 지역 드롭다운 선택 (시/도)
  2. 카테고리 칩 선택 (복수 가능)
  3. 필터 적용 → 지도가 해당 지역으로 이동 + 필터된 마커만 표시
  4. 활성 필터 개수가 필터 바에 배지로 표시
  5. "초기화" 버튼으로 모든 필터 해제
- **입력**: 지역 선택, 카테고리 선택
- **출력**: 필터된 마커, 지도 이동
- **비즈니스 규칙**:
  - 지역: 서울, 경기, 인천, 부산, 대구, 대전, 광주, 울산, 세종, 강원, 충북, 충남, 전북, 전남, 경북, 경남, 제주 (17개)
  - 카테고리: 한식, 일식, 중식, 양식, 카페/디저트, 분식, 고기/구이, 해산물, 기타 (9개)
  - 유튜버 필터 + 지역 필터 + 카테고리 필터 = AND 조건
  - 필터 결과 개수 실시간 표시 ("123개 맛집")
  - 결과 0개 시 필터 결과 개수에 "0개" 빨간색 표시
- **예외 케이스**:
  - 필터 결과 0개 → "조건에 맞는 맛집이 없어요. 필터를 조정해보세요" + 필터 초기화 버튼
  - 지역 선택 시 해당 지역에 맛집 0개 → 드롭다운에서 "(0)" 표시하되 선택은 가능
- **우선순위**: P1

### 3.4 통합 검색

- **설명**: 가게명, 지역명, 유튜버명, 카테고리를 통합 검색
- **왜 필요**: "성시경"을 검색하면 유튜버 필터, "전주"를 검색하면 지역 필터, "성게향"을 검색하면 가게 상세 — 자연스러운 UX
- **유저 시나리오**:
  1. 상단 검색바에 키워드 입력
  2. 입력 중 자동완성 드롭다운 표시 (디바운스 300ms)
  3. 결과는 카테고리별로 그룹핑: 🍽️ 맛집 / 👤 유튜버 / 📍 지역
  4. 맛집 선택 → 지도에서 해당 마커로 이동 + 인포윈도우 열림
  5. 유튜버 선택 → 유튜버 필터 적용
  6. 지역 선택 → 지역 필터 적용
- **입력**: 검색어 (텍스트)
- **출력**: 자동완성 드롭다운 → 지도 이동/필터 적용
- **비즈니스 규칙**:
  - 최소 2글자 입력 시 자동완성 시작
  - 자동완성 결과 최대 10개 (맛집 5 + 유튜버 3 + 지역 2)
  - 검색은 가게명, 주소, 유튜버명에 대해 prefix 매칭 (ILIKE '{query}%')
  - 한글 초성 검색은 MVP에서 제외 (v2)
- **예외 케이스**:
  - 검색 결과 없음 → "검색 결과가 없어요" 드롭다운
  - 검색 API 에러 → 드롭다운 숨기고, 검색바 아래에 "검색에 실패했습니다" 텍스트
  - 빈 문자열 / 공백만 입력 → 자동완성 미작동
  - 특수문자 입력 → 검색어에서 제거 후 검색 (SQL injection 방지)
- **우선순위**: P1

### 3.5 맛집 상세 페이지 (SEO)

- **설명**: 개별 맛집의 상세 정보를 보여주는 독립 페이지. 검색 엔진 유입의 핵심.
- **왜 필요**: "/restaurant/123" 같은 URL이 구글/네이버에 인덱싱되어 SEO 유입 창출
- **유저 시나리오**:
  1. 인포윈도우에서 "상세 보기" 클릭 또는 검색 엔진에서 직접 유입
  2. 가게 정보: 이름, 주소, 카테고리, 지도(소형)
  3. 방문한 유튜버 목록 + 각 유튜버의 평가/한줄평
  4. 유튜브 영상 임베드 (iframe)
  5. "네이버 지도에서 보기" 외부 링크
- **입력**: 페이지 접속 (URL)
- **출력**: 맛집 상세 정보 렌더링
- **비즈니스 규칙**:
  - 메타 태그: `<title>{가게명} - {지역} 맛집 | MukMap</title>`
  - OG 태그: og:title, og:description(유튜버 한줄평), og:image(영상 썸네일)
  - 구조화 데이터: JSON-LD Restaurant schema
  - 영상 임베드는 lazy load (뷰포트 진입 시)
  - 네이버 지도 링크: `https://map.naver.com/v5/search/{가게명} {주소}`
- **예외 케이스**:
  - 존재하지 않는 restaurant ID → 404 페이지 ("이 맛집을 찾을 수 없어요")
  - 좌표 없는 맛집 → 소형 지도 숨기고 주소만 표시
  - 영상 삭제됨 → 임베드 실패 시 "영상이 삭제되었습니다" 플레이스홀더 + 썸네일
  - 유튜버 평가/한줄평 없음 → "평가 정보가 없습니다" 텍스트
- **우선순위**: P0

### 3.6 유튜버 프로필 페이지 (SEO)

- **설명**: 특정 유튜버가 다녀간 맛집 목록 페이지. "성시경 맛집" 검색 유입용.
- **왜 필요**: "성시경 맛집 리스트"는 검색량이 높은 키워드. SEO 핵심 페이지.
- **유저 시나리오**:
  1. 유튜버 칩 클릭 → 프로필 페이지 이동, 또는 검색 엔진에서 직접 유입
  2. 유튜버 기본 정보: 채널명, 아바타, 맛집 수
  3. 맛집 리스트 (카드 형태): 가게명, 카테고리, 지역, 평가, 영상 썸네일
  4. 맛집 카드 클릭 → 맛집 상세 페이지로 이동
  5. 소형 지도에 해당 유튜버 맛집 전체 마커 표시
- **입력**: 페이지 접속 (URL)
- **출력**: 유튜버 맛집 리스트 + 지도
- **비즈니스 규칙**:
  - 메타 태그: `<title>{유튜버명} 맛집 리스트 | MukMap</title>`
  - 맛집 목록 정렬: 최신 영상 기준 내림차순
  - 페이지네이션: 무한 스크롤 (20개씩 로드)
  - 채널 썸네일은 YouTube Data API에서 가져온 것 사용
- **예외 케이스**:
  - 존재하지 않는 채널 ID → 404 페이지
  - 해당 유튜버 맛집 0개 → "아직 수집된 맛집이 없어요. 곧 업데이트됩니다!" 안내
  - 채널 썸네일 로딩 실패 → 기본 아바타 이미지 표시
- **우선순위**: P0

### 3.7 데이터 수집 파이프라인 (백그라운드)

- **설명**: 등록된 유튜버 채널의 새 영상을 자동 감지하고, 자막에서 맛집 정보를 추출하여 DB에 저장.
- **왜 필요**: 수동 입력 없이 데이터가 자동으로 쌓여야 서비스 지속 가능.
- **실행 흐름**:
  1. Railway 크론잡 (6시간마다): 등록된 채널의 최근 영상 목록 조회
  2. 이미 처리된 video_id 필터링
  3. 새 영상을 processing_queue에 등록
  4. 각 영상: 자막 추출 → Claude Haiku로 가게 정보 추출 → 네이버 로컬 검색으로 좌표 변환 → DB 저장
  5. 영상 간 2-5초 랜덤 딜레이 (rate limit 방지)
- **비즈니스 규칙**:
  - 1회 실행 시 최대 20개 영상 처리 (API 비용 제어)
  - 자막 없는 영상 → status='no_transcript'로 기록, 스킵
  - Claude가 가게명 추출 실패 (빈 배열) → status='no_restaurant'로 기록
  - 네이버 검색 실패 → 좌표 null로 저장 (지도 표시 안 됨, 리스트에는 표시)
  - 광고/협찬 영상 → is_ad=true, 별도 표시 또는 필터 가능
  - 하나의 영상에서 여러 가게 추출 가능 (각각 별도 레코드)
- **예외 케이스**:
  - youtube-transcript-api 차단 → 에러 로그 + status='api_blocked', 다음 실행 시 재시도 (최대 3회)
  - Claude API 에러 → status='llm_error', 재시도 (최대 2회, 지수 백오프)
  - 네이버 API rate limit → 429 응답 시 60초 대기 후 재시도
  - Supabase 연결 실패 → 크론잡 전체 중단, 에러 로그
  - 중복 가게 (같은 이름+비슷한 주소) → name+address UNIQUE 제약으로 UPSERT
- **우선순위**: P0

```
  └─ 직접 접속 (mukmap.kr) → 메인 지도

[메인 지도]
  ├─ 마커 클릭 → 인포윈도우 → "상세 보기" → 맛집 상세 페이지
  ├─ 유튜버 필터 → 해당 유튜버 맛집만 표시
  │   └─ 유튜버 이름 클릭 → 유튜버 프로필 페이지
  ├─ 지역/카테고리 필터 → 필터된 맛집 표시
  └─ 검색 → 자동완성 → 맛집/유튜버/지역 선택

[맛집 상세 페이지]
  ├─ "영상 보기" → 유튜브 (외부)
  ├─ "네이버 지도" → 네이버 지도 (외부)
  └─ 유튜버 아바타 클릭 → 유튜버 프로필 페이지

[유튜버 프로필 페이지]
  ├─ 맛집 카드 클릭 → 맛집 상세 페이지
  └─ 지도 마커 클릭 → 인포윈도우 → 맛집 상세 페이지
```

## 5. 데이터 모델 (개념 수준)

- **채널(Channel)**: 유튜브 채널. 이름, 썸네일, 채널 ID.
- **맛집(Restaurant)**: 가게. 이름, 주소, 좌표(위경도), 카테고리, 지역(시/도).
- **영상(Video)**: 유튜브 영상. 영상 ID, 채널(FK), 맛집(FK), 제목, 썸네일, 유튜버 평가, 한줄 요약, 광고 여부.
- **관계**: 채널 1:N 영상, 맛집 1:N 영상. 하나의 영상이 여러 맛집을 포함할 수 있음 (영상-맛집 M:N이지만, videos 테이블에 restaurant_id FK로 1:N 구현 — 같은 video_id에 다른 restaurant_id 레코드 여러 개).
- **처리 큐(ProcessingQueue)**: 크론잡 상태 관리. video_id, channel_id, 상태(pending/processing/done/failed/no_transcript/no_restaurant/api_blocked/llm_error), 에러 메시지.

## 6. 외부 의존성

| API | 용도 | 무료 한도 | 비용 (초과 시) | 장애 대응 |
|-----|------|----------|---------------|----------|
| 카카오맵 JS SDK | 지도 렌더링 | 일 30만 호출 | 무료 | 에러 메시지 + 새로고침 유도 |
| YouTube Data API v3 | 채널 영상 목록 조회 | 일 10,000 유닛 | $0 (쿼터 증가 신청 가능) | 크론잡 중단, 다음 실행 시 재시도 |
| youtube-transcript-api | 자막 추출 (비공식) | 무제한 (비공식) | 무료 | IP 차단 시 프록시 또는 대기 |
| Claude Haiku API | 가게 정보 추출 | 없음 (pay-per-use) | ~$0.25/M input tokens | 재시도 2회 후 스킵 |
| 네이버 로컬 검색 API | 가게명 → 좌표 변환 | 일 25,000 호출 | 무료 (한도 내) | 좌표 null 저장 |

**비용 추정 (월)**:
- Claude Haiku: 영상 100개 × ~2,000 토큰 = 200K 토큰 ≈ $0.05
- 나머지: 무료 한도 내

## 9. MVP 범위

**MVP 포함**:
- 지도 맛집 탐색 (마커 + 인포윈도우)
- 유튜버별 필터 (초기 5개 채널: 돌시네아, 먹을텐데(성시경), 쯔양, 히밥, 백종원)
- 지역/카테고리 필터
- 통합 검색
- 맛집 상세 페이지 (SEO)
- 유튜버 프로필 페이지 (SEO)
- 자동 수집 파이프라인 (크론잡)
- 애드센스 자리 (광고 슬롯만 배치)

**MVP 제외 (v2)**:
- 회원가입/로그인
- 즐겨찾기 서버 저장 (로컬스토리지는 가능)
- 캐치테이블/네이버 예약 연동
- 한글 초성 검색
- 유튜버 채널 자동 등록 (수동으로 DB에 추가)
- 앱 (iOS/Android)

## 10. 제약 사항 & 리스크
